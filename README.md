Introduction to Cloud Platforms (AWS Practical)
📌 Objective

The objective of this practical is to familiarize students with cloud computing platforms, specifically Amazon Web Services (AWS), and understand its interface, services, and pricing model.

🧠 Overview

Cloud computing provides on-demand access to computing resources such as servers, storage, and networking over the internet. In this practical, we explore the AWS platform and its core services.

🛠️ Steps Performed

1️⃣ Create Free-Tier AWS Account
Visited the AWS official website: https://aws.amazon.com/
Clicked on Create Free Account
Entered required details (email, password, account name)
Verified mobile number via OTP
Added payment details (for verification purposes)
Selected Basic Support Plan
Successfully logged into AWS Management Console

2️⃣ Explore AWS Dashboard

After logging in, the AWS Management Console was explored:

🔹 Key Sections Identified:
Services Menu – Access all AWS services
Search Bar – Quickly find services
Region Selector – Choose server location
Billing Dashboard – Monitor usage and costs

3️⃣ Identify Core AWS Services
💻 Compute Services
EC2 (Elastic Compute Cloud) – Virtual servers
Lambda – Serverless computing

💾 Storage Services
S3 (Simple Storage Service) – Object storage
EBS (Elastic Block Store) – Block storage for EC2

🌐 Networking Services
VPC (Virtual Private Cloud) – Private network
Route 53 – DNS management
Elastic Load Balancer – Traffic distribution

4️⃣ Understanding Pricing (AWS Pricing Calculator)
Visited AWS Pricing Calculator: https://calculator.aws/
Explored cost estimation for:
EC2 instances
S3 storage
Learned about:
Pay-as-you-go model
Free-tier limits
Monthly cost estimation

📊 Key Learnings
Understood the basics of cloud computing and AWS platform
Learned how to navigate AWS dashboard
Identified major services in compute, storage, and networking
Gained knowledge of AWS pricing and cost estimation

⚠️ Notes
AWS Free Tier has limited usage (12 months for some services)
Billing details are required but charges apply only if limits exceed
Always monitor usage in Billing Dashboard to avoid unexpected costs

📎 Conclusion

This practical helped in gaining hands-on experience with AWS cloud platform. It provided a strong foundation for understanding cloud services, navigation of the AWS interface, and cost management strategies.
